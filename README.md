# Gator

Gator is a small command-line RSS reader backed by PostgreSQL. It lets multiple
local users share a feed list, follow the feeds they care about, collect posts in
the background, and browse recent posts from the terminal.

## Requirements

- Go 1.26 or newer
- PostgreSQL
- Goose, for running the SQL migrations

Install Goose if you do not already have it:

```bash
go install github.com/pressly/goose/v3/cmd/goose@latest
```

## Install

```bash
go install github.com/akagitsunee/gator@latest
```

Make sure your Go bin directory is on your `PATH`. It is usually `~/go/bin`.

For local development from this repository, use:

```bash
go run . <command> [args...]
```

## Database Setup

Create a PostgreSQL database for Gator:

```bash
createdb gator
```

Run the migrations from the repository root:

```bash
goose -dir sql/schema postgres "postgres://postgres:postgres@localhost:5432/gator?sslmode=disable" up
```

Change the connection string to match your local PostgreSQL user, password, host,
port, and database name.

## Configuration

Gator reads `~/.gatorconfig.json`.

Create it with your database URL:

```json
{
  "db_url": "postgres://postgres:postgres@localhost:5432/gator?sslmode=disable"
}
```

The `current_user_name` field is added when you register or log in.

## Basic Flow

Register a local user:

```bash
gator register aka
```

Add a feed. The user who adds a feed follows it automatically:

```bash
gator addfeed "Y-Combinator" "https://news.ycombinator.com/rss"
```

Start collecting posts:

```bash
gator agg 1m
```

In another terminal, browse recent posts for the current user:

```bash
gator browse 10
```

## Commands

| Command | Description |
| --- | --- |
| `gator register <name>` | Create a user and make them current. |
| `gator login <name>` | Switch to an existing user. |
| `gator users` | List users. The current user is marked. |
| `gator reset` | Delete all users. Feed data is removed through cascading deletes. |
| `gator addfeed "<name>" "<url>"` | Add a feed and follow it as the current user. |
| `gator feeds` | List all feeds, their URLs, and who added them. |
| `gator follow "<url>"` | Follow an existing feed by URL. |
| `gator following` | List feeds followed by the current user. |
| `gator unfollow "<url>"` | Stop following a feed. |
| `gator agg <duration>` | Fetch one feed at a time on an interval, such as `30s`, `1m`, or `1h`. |
| `gator browse [limit]` | Show recent posts from followed feeds. Defaults to 2 posts. |

Commands that add, follow, unfollow, list followed feeds, or browse posts require
a current user. Use `register` or `login` first.

## Development Notes

The database layer is generated by `sqlc` from `sql/queries`. If you change the
queries or schema, regenerate the Go code with:

```bash
sqlc generate
```
